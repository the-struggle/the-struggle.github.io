Python and Automated Trading.

Aug 9, 2018

Zut! I missed a day! This was because a possible job fell through so I spent yesterday submitting applications. Now, back to research and work! Today I am thinking a bit about what software to use. I like Python, so will be doing my work in that language.

In general, most of the work in developing a strategy will be done in Pandas, exploring data and different possible statistical models, possibly incorporating Numpy and Scipy. Once we have a strategy, though, we need to backtest and execute that strategy using some algorithmic trading library. I see several algorithmic trading tools currently available:

  • Zipline
  • Backtrader
  • PyAlgoTrade
  • Oanda
  • QTPyLib

These seem like the most popular (or interesting). An important feature for me is that the backtesting and trading be done through the same interface, which these (or at least those that I have already looked at) do. Beyond that, the different features and syntax are all particular. Because of that I think a fun exercise would be to use a simple strategy and try them all out! I’ll try to make this my goal. Tomorrow I will discuss a bit a simple moving average crossover strategy that I’ll use as my working example. See you then!
